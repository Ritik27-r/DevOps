# AWS S3 (Simple Storage Service)

## What is S3?

Amazon S3 is an **object storage service** used to store and retrieve any amount of data from anywhere in the cloud.

It is highly durable, scalable, and secure.

---

## Why S3 is Used

* Store files (images, videos, backups)
* Host static websites
* Data backup and recovery
* Log storage
* Big data analytics storage

---

## S3 Core Concepts

### 1. Buckets

A bucket is a **container for storing objects**.

Example:

```text id="s3_bucket_01"
my-app-bucket
user-images
backup-data
```

Bucket names must be globally unique.

---

### 2. Objects

Objects are the actual files stored in S3.

Each object has:

* Data (file content)
* Metadata
* Key (file name/path)

Example:

```text id="s3_object_01"
photo.jpg
video.mp4
backup.zip
```

---

### 3. Keys

A key is the **full path of the object inside a bucket**.

Example:

```text id="s3_key_01"
folder1/image.png
logs/2026/app.log
```

---

## S3 Structure

```text id="s3_structure_01"
Bucket → Folder → Object
```

Technically, folders are just part of the key name.

---

## S3 Storage Types (Classes)

### 1. Standard

* Frequent access
* High performance

---

### 2. Intelligent-Tiering

* Automatically moves data between tiers
* Cost optimization

---

### 3. Standard-IA (Infrequent Access)

* Lower cost
* Slight retrieval fee

---

### 4. Glacier

* Archival storage
* Very low cost
* Slow retrieval

---

## S3 Features

* 99.999999999% durability (11 9’s)
* Unlimited storage
* Highly scalable
* Secure access control

---

## Uploading Files to S3

Using AWS CLI:

```bash id="s3_upload_01"
aws s3 cp file.txt s3://my-bucket/
```

---

## Downloading Files

```bash id="s3_download_01"
aws s3 cp s3://my-bucket/file.txt .
```

---

## Listing Bucket Contents

```bash id="s3_list_01"
aws s3 ls s3://my-bucket/
```

---

## S3 Access Control

### 1. IAM Policies

Control user access.

### 2. Bucket Policies

Control bucket-level permissions.

### 3. ACLs (legacy)

Less commonly used now.

---

## Public vs Private Buckets

### Private (default)

* Only authorized users can access

### Public

* Accessible via URL

Example:

```text id="s3_public_01"
https://bucket-name.s3.amazonaws.com/file.jpg
```

---

## S3 Versioning

Keeps multiple versions of an object.

Benefits:

* Prevent accidental deletion
* Restore old versions

---

## S3 Lifecycle Rules

Automatically manage storage:

Example:

```text id="s3_lifecycle_01"
30 days → Standard-IA
90 days → Glacier
365 days → Delete
```

---

## S3 Encryption

### Types:

* SSE-S3 (AWS managed)
* SSE-KMS (Key Management Service)
* Client-side encryption

---

## Static Website Hosting

S3 can host websites directly.

Example:

* HTML
* CSS
* JavaScript

## S3 Use Cases

## S3 vs EBS vs EFS

| Feature  | S3             | EBS           | EFS          |
| -------- | -------------- | ------------- | ------------ |
| Type     | Object storage | Block storage | File storage |
| Access   | Internet/API   | EC2 only      | Multiple EC2 |
| Use case | Files          | OS disks      | Shared files |

---

## Common S3 Mistakes

* Making bucket public accidentally
* Not enabling versioning
* Poor lifecycle management
* Not encrypting sensitive data
* Ignoring access policies

---

## Best Practices

* Keep buckets private by default
* Enable versioning
* Use lifecycle policies
* Enable encryption
* Use IAM roles for access
