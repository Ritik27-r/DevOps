# Docker Volumes

## What are Docker Volumes?

Docker volumes are used to **persist data generated by containers**.

Without volumes, container data is lost when the container is removed.

---

## Why Volumes are Important

* Persistent storage for containers
* Data survives container deletion
* Easy backup and migration
* Shared storage between containers
* Recommended for databases

---

## Types of Docker Storage

### 1. Volumes (Recommended)

Managed by Docker.

Stored in:

```text id="v1o2l3"
/var/lib/docker/volumes/
```

---

### 2. Bind Mounts

Directly maps host directory to container.

---

### 3. tmpfs Mounts

Temporary storage in memory (not persistent).

---

## Docker Volumes Basics

### Create a volume

```bash id="c1r2e3"
docker volume create my_volume
```

---

### List volumes

```bash id="l1v2o3"
docker volume ls
```

---

### Inspect volume

```bash id="i1n2s3"
docker volume inspect my_volume
```

---

### Remove volume

```bash id="r1m2v3"
docker volume rm my_volume
```

---

## Using Volumes in Containers

### Attach volume

```bash id="r1u2n3"
docker run -d -v my_volume:/app nginx
```

Explanation:

* `my_volume` → Docker volume
* `/app` → container directory

---

## Bind Mount Example

```bash id="b1i2n3"
docker run -d -v $(pwd):/app nginx
```

This maps:

* Host current directory → container `/app`

---

## Difference: Volume vs Bind Mount

| Feature           | Volume         | Bind Mount           |
| ----------------- | -------------- | -------------------- |
| Managed by Docker | Yes            | No                   |
| Location          | Docker storage | Host filesystem      |
| Portability       | High           | Low                  |
| Recommended       | Yes            | Only for development |

---

## Persistent Data Example (Database)

```bash id="m1y2s3"
docker run -d \
  -v db_data:/var/lib/mysql \
  mysql
```

---

## Volume Sharing Between Containers

```bash id="s1h2a3"
docker run -d -v shared_data:/data container1
docker run -d -v shared_data:/data container2
```

Both containers share same data.

---

## Removing Containers with Volumes

By default, volumes are NOT removed.

To remove volume with container:

```bash id="r1m2a3"
docker rm -v container_id
```

---

## Pruning Unused Volumes

Remove unused volumes:

```bash id="p1r2u3"
docker volume prune
```

---

## Inspecting Volume Data

Find volume location:

```bash id="i1s2e3"
docker volume inspect my_volume
```

---

## tmpfs Mount Example

Stores data in RAM:

```bash id="t1m2p3"
docker run --tmpfs /app nginx
```

* Fast
* Not persistent
* Used for sensitive temporary data

docker run -d \
  --name mysql-db \

  mysql

## Data Persistence Flow

```text id="d1p2f3"
Container → Writes data → Volume → Survives container deletion
```

---

## Common Mistakes

* Not using volumes for databases
* Storing data inside container filesystem
* Forgetting to mount volume
* Deleting volumes accidentally
* Using bind mounts in production

---

## Best Practices

* Always use volumes for persistent data
* Avoid storing data inside container layers
* Use named volumes instead of anonymous ones
* Regularly backup important volumes
* Use bind mounts only for development

---

## Security Considerations

* Restrict access to volume directories
* Avoid exposing sensitive host directories
* Encrypt sensitive data if needed
* Use proper permissions for mounted data

---

## Summary

Docker volumes provide:

* Persistent storage
* Data safety
* Shared access between containers
* Better production reliability
